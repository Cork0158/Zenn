## 解説
とりあえず、なにかの暗号が使われているのだろうということで色々と試してみましたがダメでした。そのため、一度観察してみようと思い文字列を見ていると綺麗な区切りで改行されていることに気づきました。 なんか見たことがあるなと思い、調べてみると[Base64](https://ja.wikipedia.org/wiki/Base64)で変換された文字列っぽいということがわかりました。 （これを解いた2ヶ月ほど前にメールが転送されてきたのですが、転送されたメールの内容がBase64で変換されたものが添付されていて、decodeしたので偶然見覚えがありました。） 
文字列を変換するときにとても便利なツールである、[CyberChef](https://gchq.github.io/CyberChef/)を用いてBase64でdecodeしました。 ![](https://storage.googleapis.com/zenn-user-upload/5c4d492c2977-20211209.png =500x) *CyberChef* 
ただ、一度decodeしただけではあまり文字列に変化がなかったため何度かdecodeした結果、16回目でそれっぽい文字列が出てきました。 

``` 
begin 666 
51DQ!1U]&94QG4#-3:4%797I74$AU 
end 
``` 

よくわからんけど2行目がFLAGっぽいだろうということで`begin 666`で検索してみると、[uuencode](https://ja.wikipedia.org/wiki/Uuencode)で変換されたものだろうということでdecodeしてくれるサイトを探して、decodeしてみると、FLAGを得ることができました。

 ```
 FLAG_FeLgP3SiAWezWPHu
 ```
